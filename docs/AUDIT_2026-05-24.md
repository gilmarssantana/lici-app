# Auditoria operacional LICI - 2026-05-24

## Resultado

Status geral: operacional, com um alerta controlado de segurança.

## Funciona

- `lici-api.service`: ativo.
- `lici-memory.service`: ativo.
- `lici-frontend.service`: ativo.
- Timers: backup, healthcheck, scheduler e log rotate ativos.
- Backend compila com `python3 -m compileall`.
- Frontend compila com `npm run build`.
- Rotas públicas principais retornam `200`:
  - `/health`
  - `/health/full`
  - `/decisao/engine`
  - `/edital/analyzer`
  - `/triagem/engine`
  - `/scheduler/status`
  - `/observabilidade/status`
- Rotas protegidas retornam `401` sem token, como esperado.
- Login JWT do usuário `liciadmin` foi recuperado e validado internamente.
- Rotas autenticadas principais retornam `200` com token:
  - `/auth/me`
  - `/dashboard/comando`
  - `/dashboard/resumo`
  - `/dashboard/casos`
  - `/documental/engine`
  - `/ia-assistiva/engine`
  - `/casos`
- Backup mais recente abre em dry-run.
- Dump PostgreSQL passa em `gunzip -t`.
- Manifesto de backup aponta para archive e dump existentes.
- Git local está sincronizado com GitHub.

## Corrigido nesta auditoria

- Credencial bootstrap antiga estava inválida para login.
- Criado `scripts/reset_admin_password.py` para recuperar usuário admin sem expor senha em Git/log.
- Senha de recuperação do `liciadmin` foi salva localmente em arquivo protegido:
  - `/root/lici-app/auth/admin_recovery_credentials`
- Credencial bootstrap antiga foi removida.
- `GET /health/full` agora alerta quando existir credencial de recuperação admin pendente.
- README atualizado com procedimento de recuperação admin.

## Alertas restantes

- `auth:admin_recovery_credentials`: alerta esperado.

Motivo: existe uma credencial temporária de recuperação admin. Ela deve ser usada para login, depois a senha deve ser rotacionada e o arquivo removido.

## Não foi exposto

- Nenhuma senha foi versionada.
- Nenhum token foi versionado.
- Nenhum arquivo em `auth/` ou `secrets/` foi enviado ao Git.
- Dados operacionais, documentos, memória viva, logs e banco continuam fora do repositório.

## Próximos passos recomendados

1. Usar a credencial de recuperação para entrar como `liciadmin`.
2. Trocar a senha por uma senha definitiva.
3. Remover `/root/lici-app/auth/admin_recovery_credentials`.
4. Rodar `/health/full` e confirmar status sem alerta de credencial.
5. Configurar cópia externa automática de `/root/backups/lici`.
